# 2026-03-23 Status

## Summary

Alice now has a real local MVP loop running in-house:

- `hub-api` running on FastAPI + SQLite
- local Mosquitto broker
- real ESP32 sensor node publishing temperature, illuminance, and motion
- real ESP32 relay/light node receiving commands and publishing state + ack
- web dashboard for login, stack health, devices, entities, device detail, relay control, and auto-light settings
- assistant runtime with session history and Ollama-backed planning fallback to deterministic Home OS tools

This is no longer just scaffolded architecture. The system has been exercised end to end against real boards in the house.

## What Works

- local admin login against Home OS
- device/entity projection from MQTT `hello`
- retained-state rehydration after hub restart
- MQTT `availability`, `state`, `telemetry`, and `ack` ingest
- relay command publish through Home OS only
- relay command acknowledgements visible in Home OS and dashboard
- auto-light rule driven by the light sensor and persisted in SQLite
- assistant runtime tool calls through Home OS
- Ollama planner working with `qwen3:4b`
- OTA prototype support on both current ESP32 boards

- manual `EN/RST` is now a debugging aid, not a required operating step
